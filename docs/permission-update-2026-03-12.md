# Agent 权限矩阵优化记录

**日期**: 2026-03-12  
**变更类型**: 权限收紧优化 + 遗留清理  
**影响范围**: 门下省 (Menxia) 权限、太子角色移除

---

## 变更摘要

### 核心变更
收紧门下省的 subagent 调用权限，强化"中书省作为唯一调度中枢"的架构设计。

### 具体修改

| 配置项 | 修改前 | 修改后 |
|--------|--------|--------|
| `menxia.allowAgents` | `["shangshu", "zhongshu"]` | `["zhongshu"]` |

### 权限流向对比

**修改前:**
```
门下省准奏 → 直接调用尚书省执行
```

**修改后:**
```
门下省准奏 → 返回中书省 → 中书省决定是否调用尚书省
```

## 设计原则

### 为什么要收紧门下省权限？

1. **强化中书省的调度中枢地位**
   - 门下省只负责审议，不负责调度
   - 准奏/封驳的结果统一由中书省处理

2. **符合古代三省六部制度设计**
   - 历史上门下省审议后返回中书省
   - 由中书省呈报皇帝或转尚书省执行

3. **避免权限分散**
   - 防止多个 Agent 都能直接调用尚书省
   - 统一入口便于监控和审计

## 权限矩阵（更新后）

| From ↓ \ To → | 中书 | 门下 | 尚书 | 六部+吏 |
|:---:|:---:|:---:|:---:|:---:|
| **中书省** | — | ✅ | ✅ | |
| **门下省** | ✅ | — | | |
| **尚书省** | ✅ | ✅ | — | ✅ |
| **六部+吏部** | | | ✅ | — |

### 权限说明

| 部门 | 可调用 | 说明 |
|------|--------|------|
| **中书省** | 门下省、尚书省 | 唯一调度中枢，可提交审议或直接派发 |
| **门下省** | 中书省 | 仅返回审议结果，不可直接调用尚书省 |
| **尚书省** | 中书省、门下省、六部+吏部 | 统筹执行，可向上反馈或向下派发 |
| **六部+吏部** | 尚书省 | 仅回传执行结果，不可跨部门通信 |

## 向后兼容性

### 对现有部署的影响
- **新安装**: 自动使用新权限配置
- **已安装**: 需要重新运行 `install.ps1` 或手动修改 `openclaw.json`

### Agent 行为变化
- **门下省**: 无需修改 SOUL.md 工作流（原本就不直接调用尚书省）
- **中书省**: 需确保准奏后主动调用尚书省（已在 SOUL.md 中明确）
